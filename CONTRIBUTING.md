# Contributing

Steps on how to contribute

## Reporting Issues

Before creating a new issue, please check the existing issues in this repository to avoid duplicates.

If it hasn’t been reported, please include the following:

### 1. Explain the Issue

* What did you expect to happen?
* What actually happened?

### 2. Reproduction Steps

If possible, explain how to recreate the issue step-by-step/what you were doing when it happened.

### 3. Media (Not Required)

Provide visual evidence or showcase the steps to recreate when possible:

* Screenshot
* GIF
* Video


---

## Contributing Code

* Keep changes focused and minimal
* Follow the existing code style
* Test your changes before submitting

### Pull Requests

When submitting a pull request:

* Clearly describe what your changes do
* Reference related issues (if applicable)
* Ensure your code works as expected
